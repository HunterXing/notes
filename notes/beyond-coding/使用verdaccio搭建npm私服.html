使用verdaccio搭建npm私服

vuePress-theme-reco 邢走在云端    2023

使用verdaccio搭建npm私服

邢走在云端 2022-06-26 13:50:11 npmverdaccio组件

在公司搭建npm的私服，可以保证公司的组件安全，员工内网访问也比较快速。

在这里推荐verdaccio，特别简单好用

# 一、Verdaccio 框架

Verdaccio 是一个 Node.js 创建的轻量的私有 npm proxy registry。

  • 基于 Node.js 的网页应用程序
  • 私有 npm registry
  • 本地网络 proxy
  • 可插入式应用程序
  • 易安装和使用
  • 提供 Docker 和 Kubernetes 支持
  • 与 yarn, npm 和 pnpm 100% 兼容
  • forked 于 sinopia@1.4.0 并且 100% 向后兼容。

# 二、内网安装和使用verdaccio

注意下面提到的 http://192.168.xxx.xxx 均为自己内网的服务器ip地址

# 2.1 Nodejs 环境下全局安装 verdaccio

npm install -g verdaccio

# 2.2 修改配置文件

打开默认启动的 config.yaml 文件：

vim /root/.config/verdaccio/config.yaml

👇

auth:
  htpasswd:
    file: ./htpasswd
    # Maximum amount of users allowed to register, defaults to "+inf".
    # You can set this to -1 to disable registration.
    max_users: -1
uplinks:
  npmjs:
    url: https://registry.npmjs.org/
  cnpm:
    url: https://registry.npm.taobao.org
packages:
  "@aemp/*":
    access: $authenticated
    publish: $authenticated
    unpublish: $authenticated
  "@*/*":
    access: $all
    publish: $authenticated
    unpublish: $authenticated
    proxy: cnpm
  "**":
    access: $all
    publish: $authenticated
    unpublish: $authenticated
    proxy: cnpm
listen: 0.0.0.0:4873

access 是访问权限控制，总共有三种身份：所有人($all)、匿名用户($anonymous)、认证(登陆)用户($authenticated)。

在最下方添加 对外暴露的端口

listen: 0.0.0.0:4873
  • storage：#所有包的缓存目录 建议修改如：D:\npmPackes\storage
  • proxy:代理 表示没有的包verdaccio库会去这个npmjs (即配置中https://registry.npmjs.org/) 或 cnpm 里面去找.

# 2.3 启动

verdaccio

如果 verdaccio不能直接执行, 如

-bash: verdaccio: 未找到命令

是因为没有自动注册到全局的命令，需要映射下，格式如下，命令中具体位置得看自己的实际情况

ln -s /usr/src/node-v12.13.0-linux-x64/bin/pm2 /usr/local/bin/

这时候在内网中访问 http://192.168.xxx.xxx:4873 就可以访问了。

但是情况我们关闭命令行窗口之后，服务就停止了，那么我们就需要使用pm2进行进程守护。

# 三、使用pm2进程守护

# 3.1 安装和使用

pm2的使用可以看我19年整理的一个博客 👉利用pm2进行node的进程守护

npm install -g pm2
pm2 start verdaccio

注意此时如果 pm2 未找到命令，也需要和2.3中一样使用ln进行映射

这时候关闭控制台也能让这个node服务持久化运行

# 四、用户管理

# 3.1 添加用户

npm adduser --registry http://192.168.xxx.xxx:4873

npm adduser --registry http://192.168.59.235:8062

输入 username、password 以及 Email 即可

# 3.2 登录

npm login --registry http://192.168.xxx.xxx:4873

# 3.3 上传私有包

npm publish --registry http://192.168.xxx.xxx:4873

# 3.4 其他

上述的操作每次都要指定源registry，可以使用 nrm进行快速切换 npm的镜像源

nrm add verdaccio http://192.168.xxx.xxx:4873

nrm use verdaccio

此处不赘述nrm的使用方法。
