# webpack5 详解

# 目标

使用webpack5构建vue项目

# 环境

"webpack": "^5.78.0" "webpack-cli": "^5.0.1"

# Vue2.x项目搭建

# 搭建基础项目

🎯基础项目的目标就是能成功运行App.vue中的代码

# 0. 文件结构

首先我们看vue-cli搭建的vue2 项目的结构是啥样的，我们也构建一个类似的结构, 其中多的build文件夹及其下的配置可以看成是vu e.config.js

├── build
│   └── webpack.common.js
├── package-lock.json
├── package.json
├── public
│   └── index.html
└── src
    ├── App.vue
    ├── assets
    │   └── styles
    │       └── index.css
    ├── main.js
    ├── utils
    └── views

# 1. 基本安装

（1）创建webpack5文件夹，用vscode打开，通过终端运行：

npm init -y
npm i webpack webpack-cli -D

（2）在src/main.js中写入测试代码

console.log('webpack5')

# 2. 配置出入口

build/webpack.common.js

（1）配置入口

可配置多个入口。但开发中一般是react或vue，为单页面web应用（SPA），入口一个即可

//webpack.common.js
const path = require('path')
module.exports = {
  entry: path.resolve(__dirname, "../src/main.js"),
}

（2）配置出口

只能有一个出口，这里指定输出路径为'dist'

//webpack.common.js
module.exports = {
  output: {
    path:path.resolve(__dirname,'../dist'),
    filename: '[name].bundle.js',
    clean:true //每次构建清除dist包
  },
}

现在，我们具有了最低配置。在package.json中，我们创建一个运行webpack命令构建脚本

"scripts": {
  "build":"webpack --config build/webpack.common.js",
}

现在可以运行它了:

npm run build

在dist文件夹下会生成main.bundle.js

# 3. 配置最基础的plugin和loader

# 概念
  • 插件（Plugins）是用来拓展Webpack功能的，包括：打包优化、资源管理、注入环境变量

​ 插件使用：只需要require()它，然后把它添加到plugins数组中

  • loader 用于对模块的源代码进行转换

    loader都在module下的rules中配置

    loader配置项包括：

    • test 正则校验（必须）
    • loader 调用loader的名称 / use 链式调用loader （二选一）
    • include/exclude 手动添加必修处理的文件/文件夹或屏蔽不需要处理的文件/文件夹（可选）
    • options 为loaders提供额外的设置选项（可选）

    tip：use链式调用，都是从右向左解析，需注意调用loader的顺序。loader要记住，面试经常被问到有哪些loader以及其作用

接下来安装几个基本的plugin和loader来完成这个最基础的vue项目

# html-webpack-plugin

html-webpack-plugin将为你生成一个HTML5文件，在body中使用script标签引入你所有webpack生成的bundle

（1）安装

npm i -D html-webpack-plugin

（2）编写public/index.html

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8" />
  <meta http-equiv="X-UA-Compatible" content="IE=edge" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Document</title>
</head>

<body>
  <div id="app">
    <div>webpack5</div>
  </div>
</body>

</html>

（3）配置 webpack.common.js

//webpack.common.js
const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  plugins: [
    new HtmlWebpackPlugin({
        template: path.resolve(__dirname, '../public/index.html'),
        filename: 'index.html', 
    }),
  ],
}

（4）运行

npm run build

可以看到dist下多了index.html文件，

并且打包生成的main.bundle.js在index.html中通过script标签被引入

# css-loader/style-loader

作用：加载css

css-loader：会对@import和url()进行处理

style-loader：将CSS注入到JavaScript中，通过DOM操作控制css

（1）安装

npm i css-loader style-loader -D

（2）在webpack.common.js中进行配置

module: {
    rules: [
      {
        test: /\.css$/,
        use: ["style-loader", "css-loader"], //从右向左解析
      },
    ],
  },

（3）示例

src/assets/styles/index.css

body {
    background-color: f2f2f2;
}

在main.js中引入

import './assets/styles/index.css'

重新编译 npm run build，在浏览器中打开 dist/index.html，可以看到css已生效

# vue-loader/ vue-template-compiler

  • vue-loader，是一个loader，用于解析.vue文件

  • vue-template-compiler，是plugin，用于模板编译

版本号如下

  "vue-loader": "^15.9.8",
  "vue-template-compiler": "^2.7.14"

另外需要安装，dependencies，后续在生产环境中此资源可以不打包，作为cdn引入，此处先直接作为依赖打包

npm i vue@2.7.14
# 配置 webpack.common.js
//webpack.common.js
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
// vue加载器
const { VueLoaderPlugin } = require('vue-loader')

module.exports = {
  mode: 'production',
  // 入口
  entry: {
    main: path.resolve(__dirname, "../src/main.js"),
  },
  // 出口
  output: {
    // 输出路径
    path: path.resolve(__dirname, "../dist"),
    // 输出文件名
    filename: '[name].bundle.js',
    // 每次构建清除构建文件
    clean: true
  },
  module: {
    rules: [
      // vue文件
      {
        test: /\.vue$/,
        loader: 'vue-loader',
        include: [path.resolve(__dirname, '../src')]
      },
      // css
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader', 'less-loader']
      }
    ]
  },
  // 插件
  plugins: [
    // vue加载器插件
    new VueLoaderPlugin(),
    // html模板，自动引入打包后的js文件
    new HtmlWebpackPlugin({
      template: path.resolve(__dirname, '../public/index.html'),
      filename: 'index.html'
    })
  ]
}

# 4. 编写vue代码

修改 App.vue

<template>
  <div class="app-container">
    App
  </div>
</template>

<script>
export default {
  name: 'App'
}
</script>
<style>
</style>

修改main.js

import Vue from 'vue'
import App from './App.vue'
import './assets/styles/index.less'

new Vue({
  el: '#app',
  render: h => h(App)
})

# 5. 打包项目

npm run build

可以看到dist下多了index.html文件，

并且打包生成的main.bundle.js在index.html中通过script标签被引入

部署dist即可看见vue文件内容被成功运行
