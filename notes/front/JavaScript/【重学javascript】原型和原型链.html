【重学javascript】原型和原型链

邢走在云端 2022-02-21 23:01:52 JavaScript原型原型链

# 一、class实现继承

# 1.1 实现三个类

实现动物类, 拥有种类classify这个属性和info和active两个方法

class Animal {
  constructor(classify, age) {
    this.classify = classify;
    this.age = age;
  }
  info() {
    console.log(`这是${this.classify}`);
  }
  active() {
    console.log(`${this.classify}可以活动`);
  }
}

实现人类，继承自动物，拥有姓名、年龄，以及从父类继承过来的方法和属性，其中info属性在子类中被重写

class Human extends Animal {
  constructor(name, age) {
    super(age)
    this.classify = "Human"
    this.name = name;
    this.age = age
  }
  // 重写父类info方法
  info() {
    console.log(`${this.name}今年${this.age}岁`);
  }
  // 多出thinking方法
  thinking() {
    console.log(`人类可以思考`);
  }
}

实现中国人Chinese类，继承了Human类，

class Human extends Animal {
  constructor(name, age) {
    super("人类", age)
    this.name = name;
    this.age = age
  }
  // 重写父类info方法
  info() {
    console.log(`${this.name}今年${this.age}岁`);
  }
  // 多出thinking方法
  thinking() {
    console.log(`人类可以思考`);
  }
}

实例化这些类

const reptiles = new Animal("爬行类", 200)
reptiles.info()   // 这是爬行类
reptiles.active() // 爬行类可以活动


const human = new Human("Jack", 87)
human.info()  // Jack今年87岁
human.active() // 人类可以活动


const p = new Chinese("张三", 45, "安徽")
p.farming() // 张三会种地，籍贯安徽
p.active() // 人类可以活动

reptiles 这个实例有active这个不难理解，但是 human和p可以调用active这是为什么呢？

# 1.2 隐式原型和显式原型

显式原型：prototype 隐式原型：__proto__

__proto__是每个对象都具有的属性

prototype是Function独有的属性

如图所示，p的[[prototype]]上只有 constructor和 farming，找不到active，就继续往上找一层，依旧没找到继续往上找才能找到，

js中的对象有一个特殊的[[Prototype]]内置属性,就是对其他对象的引用，几乎所有的对象在创建时候[[Prototype]]属性都会赋予一个非空值。

human.__proto__ === Human.prototype                // true
Chinese.prototype.__proto__ === Human.prototype    // true
Human.prototype.__proto__ === Animal.prototype     // true

从上不难得出一个结论： 实例的隐式原型 __protp__ 等于其对象的显示原型 prototype

p.__proto__.__proto__.__proto__=== Animal.prototype // true

要是我们顺着原型继续往上找会是啥,

p.__proto__.__proto__.__proto__.__proto__ === Object.prototype // true

再继续往上

p.__proto__.__proto__.__proto__.__proto__.__proto__   // null

也就是说 "无，名天地之始"

Object.prototype.__proto__ === null   // true

上面那个好长好长的链状关系就是所谓的原型链，如果一个对象实例在自身原型上找不到属性或方法，原型链的特性就会让它继续往上找，找不到就会 undefined

引用MDN上的一句话

JavaScript 对象是动态的属性“包”（指其自己的属性）。JavaScript 对象有一个指向一个原型对象的链。当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及该对象的原型的原型，依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾(undefined)。

# 二、总结

class其实是function的语法糖

typeof Human === "function"  // true

  • 对象的隐式原型的值为其对应构造函数的显式原型的值 fn.__proto__ === Function.prototype

    也可以说

    一个类的实例的隐式原型为该类的显式原型

  • 函数的prototype属性是定义时自动添加的。默认为{}

  • 对象的__proto__属性是创建对象时自动添加的，默认值为其构造函数的prototype Object.prototype.__proto__ === null
